# Usability Testing Checklist – Banking Application

## Clarity & Communication
- [ ] Error messages are specific and actionable (e.g., "Enter a valid 10-digit mobile number" instead of "Invalid input")
- [ ] Success confirmations clearly state what happened (amount, beneficiary, reference number)
- [ ] Financial terms (EMI, NEFT, IMPS, RTGS) are used consistently and, where helpful, briefly explained

## Efficiency
- [ ] Frequently used actions (fund transfer to saved beneficiary, bill payment) require minimal steps
- [ ] Recently used beneficiaries/billers are easily accessible
- [ ] Auto-fill / saved beneficiary details reduce repetitive data entry

## Error Prevention & Recovery
- [ ] Confirmation step (review screen) shown before irreversible actions like fund transfer
- [ ] Users can cancel/edit before final submission
- [ ] Clear recovery path if a transaction fails (retry option, support contact info)

## Learnability
- [ ] First-time users can complete registration without external help
- [ ] Tooltips or helper text are available for non-obvious fields (e.g., IFSC code, CVV)
- [ ] Navigation labels are self-explanatory (no ambiguous icons without labels)

## Accessibility & Inclusivity
- [ ] Font sizes are legible without zooming
- [ ] Works reasonably well with browser zoom up to 150%
- [ ] Critical actions are not dependent on color alone (e.g., red/green status also has text/icon)

## Trust & Transparency
- [ ] Fees/charges (if any) are clearly displayed before confirming a transaction
- [ ] Loan EMI calculations show a clear breakup (principal vs interest)
- [ ] Users can easily find transaction history and download statements
